# Prompt: Review Spec

本提示词用于让不同模型独立评审 Spec 草稿。建议至少两个模型独立评审，
管理员合并意见修订后，再把 `status` 改为 `confirmed`。

## 使用方式

将下面的提示词复制给 AI，并粘贴 Spec 草稿全文。

## 模板提示词

```text
你是一名资深技术负责人和交付评审人。请评审下面这份 Spec 草稿，
判断它是否可以作为 AI Agent 分阶段执行（包括无人值守 auto 模式）的合约。

评审维度：

1. 完整性：SPEC_TEMPLATE.md 要求的章节是否齐全，frontmatter 是否合法。
2. 决策收敛：是否还有未拍板的问题散落在正文；非目标是否清晰，能否防住范围蔓延。
3. 阶段质量：每个阶段是否可独立执行、独立验证；顺序与依赖是否合理；
   是否存在验收标准差异过大被塞进同一阶段的情况。
4. 可验证性：Verify: 是否都是机器可验命令；Manual: 是否只包含必须人工判断的项。
5. 安全边界：允许路径、密钥处理、暴露面、危险操作约束是否显式；
   auto 模式下没有人工审批兜底时是否仍然安全。
6. 风险：外部依赖、成本、不可逆操作、与现有系统的冲突。

输出格式：

## 总体结论

- 结论：可定稿 / 需修改后定稿 / 需重写
- 一句话理由。

## 问题清单

每个问题包含：编号、严重级别（blocker / major / minor）、所在章节、问题说明、建议修改。

## 建议的修改稿片段

只输出需要修改的章节的建议版本，不要重写整份 Spec。

Spec 草稿如下：

<粘贴 Spec 草稿全文>
```
