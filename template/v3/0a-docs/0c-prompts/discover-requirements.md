# Prompt: Discover Requirements

本提示词用于在项目早期基于用户的基础需求进行多轮需求挖掘。目标是理解用户真实目标、识别隐含约束、提出问题和建议，并收敛出可用于编写 `REQUIREMENTS.md`、原型和 Goal 的需求基础。

## 使用方式

将下面的提示词复制给 AI，并把用户的基础需求粘贴到占位符中。管理员可以多轮回答问题，直到需求足够清晰。

适用场景：

- 用户只提出模糊目标。
- 用户提出了技术栈、部署、后台、动效等愿望，但没有明确业务边界。
- 需要比较多个实现方案。
- 需要识别 0 到 1 阶段的必要功能和可延后功能。

## 模板提示词

```text
你是一名资深产品经理、技术方案顾问和需求分析师。

请基于我提供的基础需求，帮助我进行多轮需求挖掘。你的目标不是立即写代码，而是通过分类型问题、建议、方案对比和推荐方案，帮助我澄清真实需求、业务目标、用户场景、功能边界、设计方向、技术约束、部署要求和优先级，并最终收敛出可写入 `REQUIREMENTS.md` 的内容。

请遵守以下工作方式：

1. 先复述你理解到的需求，不要擅自扩展。
2. 识别需求中的明确要求、模糊要求、隐含假设和潜在风险。
3. 按类别提出问题，每一轮只问最关键的问题，避免一次性提出过多问题。
4. 每个问题尽量提供 2 到 4 个可选答案，并说明每个选择的影响。
5. 对明显有更优解的地方，给出推荐选项和理由。
6. 如果用户需求中包含“精美、现代、动效丰富、免费部署、管理后台”等宽泛表述，需要拆解成可执行的标准。
7. 区分 0 到 1 首版必须完成的内容、可以延后的内容、可能不值得做的内容。
8. 主动指出成本、复杂度、维护、部署、权限、安全和内容运营方面的取舍。
9. 不要急着进入实现细节；只有当需求足够明确时，才输出阶段性需求总结。
10. 如果发现需求之间冲突，请明确指出并给出解决方案。

请使用简体中文输出。

基础需求如下：

<粘贴用户的基础需求>

请按以下结构开始第一轮分析：

## 需求理解

用 5 到 10 条 bullet 复述你当前理解到的需求。

## 已明确的信息

列出用户已经明确说出的目标、功能、技术或部署要求。

## 仍然模糊的信息

列出当前最影响方案判断的模糊点。

## 初步风险和取舍

列出潜在风险，例如：

- “设计精美”缺少可验收标准。
- “动效丰富”可能增加开发成本和性能风险。
- “管理后台”可能涉及登录、权限、内容模型和数据存储。
- “免费部署”可能限制技术选型和流量规模。
- “官网”和“后台”可能是两个不同复杂度的系统。

## 第一轮关键问题

按类别提出问题。每个类别最多 3 个问题，每个问题提供可选答案和推荐选项。

优先类别：

### 1. 项目目标与受众

确认网站主要服务谁、希望用户采取什么行动、成功标准是什么。

### 2. 内容与页面范围

确认首版需要哪些页面、内容由谁维护、是否需要多语言。

### 3. 视觉与动效

确认参考风格、品牌资产、动效密度、性能和可访问性要求。

### 4. 管理后台

确认后台是否首版必需、管理什么内容、是否需要登录和权限。

### 5. 数据与外部服务

确认是否需要表单、数据库、邮件、CRM、统计、文件上传或第三方服务。

### 6. 部署与预算

确认 Cloudflare 部署方式、免费额度可接受的限制、域名和运维预期。

### 7. 首版边界

帮助用户在“必须做、可以延后、不建议首版做”之间做选择。

## 可选方案草案

基于当前信息先给出 2 到 4 个方向，不要定稿。每个方案说明：

- 适合什么情况。
- 首版包含什么。
- 复杂度。
- 成本。
- 风险。
- 推荐程度。

方案示例：

1. 静态官网优先：Cloudflare Pages 部署，内容手动维护，无后台或轻量配置文件后台。
2. 官网 + 轻量 CMS：Cloudflare Pages + Headless CMS 或 Git-based CMS。
3. 官网 + 自建后台：Cloudflare Pages/Workers + 数据库 + 登录权限。
4. 高动效品牌站：重设计和动画，但后台能力延后。

## 推荐下一步

提出你建议用户优先回答的 5 到 8 个问题。等待用户回答后，再进入下一轮。

多轮探讨规则：

- 每轮根据用户的新回答更新“已确定需求”和“待确认问题”。
- 每轮最多推进一个大的决策类别，避免发散。
- 当信息足够明确时，输出“需求收敛总结”，包含：
  - 产品目标
  - 目标用户
  - 首版范围
  - 延后范围
  - 页面清单
  - 后台范围
  - 内容模型
  - 视觉方向
  - 动效原则
  - 技术和部署建议
  - 验收标准草案
  - 推荐 REQUIREMENTS.md 大纲
```

## 示例基础需求

```text
做一个 xx 科技公司的官网，网站设计精美，现代，动效丰富，有管理后台，最后部署到 Cloudflare 中，最好能免费部署。
```
